PATH-SAFE specifications

Files to be provided

  • A FASTQ 1 file containing the forward sequencing reads.
  • A FASTQ 2 file containing the reverse sequencing reads.
  • A CSV file containing the metadata associated with sequencing the sample.

File naming convention

The base filenames should be of the form

pathsafe.[sample_id].[run_name].[extension]

where

  • [sample_id] is the sample identifier assigned by the provider,
  • [run_name] is the name of the sequencing run as given by the supplier's sequencing instrument (not an internal identifier assigned by the supplier),
  • [extension] is the file extension indicating the file type.

File name extensions

The extensions ([extension]) should be

  • 1.fastq.gz for the forward FASTQ file,
  • 2.fastq.gz for the reverse FASTQ file, and
  • csv for the CSV metadata file.

Valid characters

The [sample_id], [run_name] and [extension] must contain only

  • letters (A-Z, a-z),
  • numbers (0-9),
  • hyphens (-) or
  • underscores (_).

Buckets

Bucket names follow the general convention

pathsafe-[sequencing_org]-[platform]-[test_flag]

Metadata specification

Required

Field name Data type Description
cid text Unique identifier for a project record. Set by Onyx.
country choice Country that the sample was collected in.
published_date date (YYYY-MM-DD) The date the project record was published. Set by Onyx.
purpose_of_sampling text The purpose of the sampling.
receipt_date date (YYYY-MM-DD) The date the sample was received by the sequencing provider.
run_name hash The unique identifier assigned to the run by the sequencing instrument.
sample_date date (YYYY-MM-DD) The date the sample was taken by the sequencing provider.
sample_id hash The sequencing providers identifier for the sequenced sample.
sequence_host_org choice
sequence_org choice Laboratory within the organisation the sample has been sequenced by. Codes will be provided by CLIMB-TRE. This must match the sequence_org bucket files were submitted to.
site choice
source_type choice Source of the sample.
submit_org choice Organisation or agency the sample has been submitted by.

Optional

Field name Data type Description
assembly text
county choice County the sample was collected in, taken from the ONS shapefile.
environmental_location text What type of location did the sample come from.
food_type text Type of food the sample was taken from.
host_species integer NCBI Taxonomy ID for the host species.
infection_status text Infection status of the host at the time of sampling.
pathogenwatch_uuid text
sample_details text Details relevant to interpretation of the sample.
travel_associated_dom bool Is this case associated with domestic travel?
travel_associated_intl bool Is this case associated with international travel?
travel_location_dom choice Location of domestic travel.
travel_location_intl choice Location of international travel.
